
Jahongir Travel — Tour Detail Page Review (UI/UX, SEO, Performance)
====================================================================

Summary
-------
This file lists actionable recommendations to improve UI/UX, SEO, structured data, and performance for the “Samarkand City Tour” detail page. Follow these to reach Lighthouse 90+ in Performance/SEO/Accessibility/Best Practices and improve conversions.

1) UI/UX & Conversion
---------------------
- Add four core blocks users expect and link them from a mini TOC under the title:
  • Itinerary (timeline style: 3–5 stops with approximate times)
  • Meeting Point & Pickup (Google Map embed + pickup radius / fees)
  • Price & Discounts (child pricing, private group, seasonal)
  • Know Before You Go (dress code, cash/ATM, photography rules, steps)
- Insert a compact CTA bar under Overview: [Check availability] [WhatsApp] [Call].
- Mobile: keep sticky price bar (present) and add a second sticky button “WhatsApp” next to “Book Now.”
- Gallery:
  • Open thumbnails in an accessible lightbox (keyboard esc/arrow navigation).
  • Enable swipe on mobile; ensure focus trapping in the lightbox.
  • Maintain aspect ratios and declare width/height to avoid CLS.
- Trust & reassurance near the booking card:
  • Add badges: “Secure payments”, “24/7 support”, “Free cancellation (24h)”. 
  • Add short microcopy under total: “No payment now — pay after confirmation” (or your true rule).
- Forms:
  • Date min = today + 1; validate guests ≤ maxGuests; disable Request button until valid.
  • Add inline error messaging and aria-live region for validation feedback.
- Accessibility:
  • Keep visible focus outlines; don’t remove `outline` in CSS.
  • All decorative icons: `aria-hidden="true"`. Action icons must have visible text.
  • `nav__toggle` must toggle `aria-expanded` and focus first menu item when opened.
  • Ensure color contrast ≥ 4.5:1 for all text on buttons and overlays.

2) Information Architecture & Content
-------------------------------------
- Heading hierarchy: H1 (tour title) → H2 (Overview/Highlights/Included/Cancellation/FAQ/Itinerary/Meeting Point) → H3 (subsections like Included/Not Included).
- Above-the-fold copy: keep 300–600 words of unique, scannable text. Use short paragraphs and bullets.
- Internal links in body text:
  • /cities/samarkand/
  • /insights/ articles (Registan, Shah-i-Zinda, Bibi-Khanym guides)
  • /about/ and /contact/ for trust & conversion.
- Add concise persona cues:
  • “Perfect for: history lovers, short-stay visitors, families 6+.”
  • Accessibility note: stairs at Shah-i-Zinda, uneven surfaces, etc.

3) SEO: Head & SERP Enhancements
--------------------------------
- Add/confirm canonical:

- Add OG enhancements:

  <meta property="og:locale" content="en_US">
- Remove legacy header:

- Keep unique title and meta description (already good). Avoid keyword stuffing.
- Multilingual support (if pages exist):
  • Add hreflang alternates for /ru/ and /fr/ versions.
- Breadcrumbs:
  • Keep both visible breadcrumbs and BreadcrumbList JSON-LD (already present).

4) Structured Data
------------------
- You can keep `FAQPage` JSON-LD (valid). 
- Prefer `Tour` (or keep `TouristTrip`) with `Offer`. Include:
  • price, priceCurrency, availability, priceValidUntil
  • provider Organization with logo/telephone
  • itinerary as ItemList with TouristAttraction items.
- IMPORTANT: Only include `aggregateRating` and `review` in JSON-LD if the same rating/reviews are displayed on the page to meet Google’s policy.
- Keep URLs absolute and images HTTPS; ensure images are 1200px+ for rich sharing cards.

5) Performance (Core Web Vitals)
--------------------------------
Targets: LCP < 2.5s, CLS ≈ 0, INP < 200ms.

- CSS:
  • Inline critical CSS (~10–14 KB) for above-the-fold; load rest as regular stylesheet (or `media="print" onload="this.media='all'"` pattern).
  • Minify and hash file names; serve with long-cache headers.
- Fonts & Icons:
  • Drop Font Awesome (heavy, render-blocking). Use inline SVG icons or a tiny custom icon set.
  • If using webfonts, add `font-display: swap` and preload the primary font.
- Images:
  • Hero image: do NOT lazy-load; set `loading="eager"` and `fetchpriority="high"`. Provide `srcset` + `sizes` responsive variants.
  • Thumbnails and non-critical images: `loading="lazy"` with explicit `width`/`height` to avoid layout shifts.
  • Use modern formats (WebP/AVIF). Keep compression ~85–90% quality visually.
- JS:
  • Defer non-critical scripts. Avoid inline `style="display:none"` to control load; use a CSS `is-loading` class toggled by JS.
  • Add GA4 events for `check_availability`, `request_booking`, `whatsapp_click`.
- Server:
  • Enable Brotli/Gzip compression, HTTP/2 or HTTP/3, and proper cache-control for static assets.
  • Serve images via CDN with caching if available.

6) Booking Widget UX
--------------------
- Clarify confirmation type: “Instant confirmation” or “Reply within 2–24h” (choose the real one).
- Validate guest count and date before enabling the Request button; show accessible error text.
- Display price breakdown updates live as guests change (already designed — ensure calculation and currency format).
- Provide alternative contact actions: WhatsApp deep link and a visible phone link; repeat at bottom of page.

7) Code Hygiene & A11y Details
------------------------------
- Buttons: ensure all actionable elements are <button type="button"> (you already do).
- External links: add `rel="nofollow"` for purely commercial outbound links where appropriate; always keep `rel="noopener noreferrer"`.
- Gallery visibility: avoid inline `style="display:none"`; apply a CSS class like `.is-hidden` and toggle with JS to prevent FOUC.
- Maintain consistent aria-labels; do not duplicate labels when visible text exists.
- Keep keyboard focus states on modals/lightboxes and return focus to the trigger on close.
