Quick fixes you can apply

Don’t cancel vertical scroll on the tabs strip
If you added a wheel/touch handler, allow default behavior when deltaY is present:

// example safe wheel handler for horizontal tab scrollers
const scroller = document.getElementById('sectionScroller');
scroller?.addEventListener('wheel', (e) => {
  // Only translate horizontal scroll; let vertical pass through
  if (Math.abs(e.deltaX) > Math.abs(e.deltaY)) {
    scroller.scrollLeft += e.deltaY || e.deltaX;
    e.preventDefault();
  }
}, { passive: false });


CSS to reduce scroll trapping

/* Ensure the tab strip doesn’t trap page scroll */
.section-nav__scroller {
  overflow-x: auto;
  overflow-y: visible;           /* don’t create an inner scrolling context */
  overscroll-behavior-x: contain;/* keep horizontal bounce tidy without blocking vertical */
  -webkit-overflow-scrolling: touch;
}
If you have a “no-scroll” body lock for the mobile menu
Many sites add body.no-scroll { overflow:hidden; } when the burger menu opens and forget to remove it. Make sure that class is removed when the menu is closed.

Why I’m confident it’s this

The horizontal section navigation appears immediately before the Highlights block in your DOM (the anchors for “Overview / Highlights / Included / …” live inside #sectionScroller), which is precisely where your scrolling stops. That element is the usual culprit for scroll lock on pages with tabbed/horizontal scrollers.
